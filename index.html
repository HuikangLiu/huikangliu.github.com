#编程技能

  • C程序的入口是main()，返回值类型和参数均可选，如果是使用命令行参数，可以使用这个格式
    <optional return value type>main(int argc,char *argv[])

  • 函数第一行称为函数签名，观察这个签名可以得知该函数执行后是否有返回值以及在函数内使用的形参

  • 变量的主要类型有int,float,double,char，其典型大小分别分4/2，4，8，1字节

  • 程序编译后，大多数变量都会被预分配固定大小内存。实际大小可能会受硬件影响，而 sizeof()所获取的只是由编译器分配的

  • 最常见的赋值语句是destination = source<with optional operators>

  • printf最多的使用是直接打印与借助格式化字符串打印。分别为printf(<string>)与printf(<format string>,<list of variables/values>).其中，\n为换行，%d为十进制数，%s为字符串值，%x为十六进制数

需要注意的是如果变量／值的个数与格式化符号字符串中的符号数不一致，会出现问题，将在Gray Hat 11种讨论

  • scanf的作用是获取用户输入，&表示把值存入后面变量所代指的内存地址处

  • strcpy&strncpy,strcpy的问题在于没有做边界检查，导致可能出现缓冲区溢出。前者甚至可以被称之为C语言中最危险的命令

  • 此外，做边界检查的时候，要保证宽度的正确

  • 注释使用// & /**/

  • gcc的使用，如要编译test.c
    gcc -o test.c -> ./test

  • RAM的数据在计算机活跃的时候随时可以获取到，这也是RAM的名称由来，但是在计算机关闭时，RAM内的数据会全部丢失

  • 现代处理器基于intel技术的产品(x86),内存按32位寻址，也就是说处理器用于选取特定内存地址的地址总线是32位宽，最大可寻址4 294 967 295字节

  • 字节序意味着存储数据时应该由低位字节还是高位字节存入，intel占边低位，摩托罗拉相反

＊ 内存中的程序：加载入内存后，进程会分成很多小节，以下六种为主要的节

1. .text节 基本与二进制可执行文件的.text部分保持一致，主要包含完成任务需要执行的机器指令。是只读的！如果尝试写入信息会导致段错误，其大小在运行时是固定的，当进程初次载入内存时就已经确定了。

2. .data节 用于存储已初始化的全局变量，同样也是固定大小的 int a = 1;

3. .bss节 below stack section栈下节，用于存储未初始化的全局变量，如int a;

4. 堆节 用于存储动态分配的变量，并且分配的存储空间使用的是低位字节序。内存分配通过malloc()和free()控制。

 int i = malloc(sizeof(int)):

5. 栈节  用于(递归地？)记录函数调用，在大多数系统中是高地址内存存到低地址内存的生长方式，本地变量也在此节中。而这种生长方式也导致了缓冲区溢出的存在

6. 环境／参数节 存储可能用到的系统级变量的副本，如可访问路径，shell名，主机名等

  • 缓冲区是可以接受和存放数据的存储区域，数据会保留到有进程对他进行处理为止

  • 每个进程都有自己的一组缓冲区,所以关键是让他们保持连续 ，在进程内存的.data&.bss中分配内存就可以实现这一点。缓冲区一旦分配，大小也就固定了

  • 字符串实际就是内存中连续的字符数组

＊ 指针，在内存中移动数据是缓慢的，所以需要指针。内存地址的长度是32位，所以指针的大小被设定为4字节。同时要注意，如果char *str，该指针会被存放进.bss节，因为命令里没有设定字符串的大小

####下面是一个关于内存知识的示例程序

//memory.c int index = 5; char *str; int nothing; void funct1(int c){ int i = c; str = (char*)malloc(10*sizeof(char)); strncpy(str,"abcde",5); } void main(){ funct1(1); }

#道德黑客与法律制度

  • 任何安全从业者都不应当嘲笑客户，或者让客户觉得自己对安全的认识太过于匮乏。

  • 渗透测试的团队应当尽可能少的甚至不读取任何敏感数据，以避免后续纠葛，插一句见世纪佳缘案。

  • 攻守是不对等的

  • 漏洞评估跟渗透测试不可混为一谈，渗透测试是测试人员进攻直到占领所有资源，即root权限或取得所在网络的最高管理员账户的控制权。而漏洞评估更像是对目标系统的扫描，及在漏洞库中匹配。

  • 基本规则：

    1. 设置测试人员与客户期望以及联系方式
    2. 确定测试相关方
    3. 设置开始终止日期及管制期
    4. 获取正式批准并签订关于适用范围，签名和法律要求的书面协议（通常称为工作申明SOW）

  • 被动扫描与主动扫描的区分关键在于 渗透测试人员与目标间是否为零接触。比如通过社交网络，谷歌等方式就是被动扫描，而使用扫描器，流量嗅探就是主动扫描。

  • 通常nmap就可以做到识别os，补丁，端口之类。

  • 硬化系统，这个概念很有趣，就是类似比赛中flag争夺的时候拿到一个组的分之后，主动为他加固系统，从而使别人无法攻破，最终被自己圈养的行为。当发现系统中出现了本没有进行过的操作或更新也要注意。

  • 美国USC1029法案中通过可接入设备非法访问获利超过1000美元判10年及1w美元＋双倍损失。再犯20年。我国的网络安全相关法律应该以后也会借鉴国外经验，目测是越来越严，不存侥幸心理，用在正道上才叫技术。

  • 社会中对黑客概念的误解来源于公知或者说所谓主流媒体的推波助澜。作为安全从业者要了解进攻方的实用工具，但不代表脚本小子就可以称之为安全从业者。

  • 安全是代码相关，但不是只靠代码就能保证的。 例见大学期间写的snsmap工具。

安全世界观，经验之谈。

  1. 已知目标活动的社交网站和具体用户名的情况

#####以百度贴吧为例，主要的四个步骤为：

  • 构造URL
  • 获取下一页URL
  • 判断是否存在下一页
  • 爬取当前页所有用户

###1.所要爬取的用户的初始url,以及x,其中,0是关注,1是粉丝,返回值是关注或粉丝的第一页url

def GetFF(url,x):
        response = urllib2.urlopen(url)
        html = response.read()
        soup = bs(html,'html.parser')
        link_follow = soup.find_all("span",attrs={'class':'concern_num'})
        soup = bs(str(link_follow),'html.parser')
        link_raw = soup.find_all('a',attrs={'target':'_blank'})
        link = []
        for i in link_raw:
            # print type(i)
            link.append(i.get('href'))
        url_qub = "http://tieba.baidu.com" + link[x]
        return url_qub
**[参数里的url可以由固定url前部加用户名构造，其中中文可以使用urllib.quote()进行传参]** ###2.输入某url,返回下一页url 
def PageNum(url):
    response = urllib2.urlopen(url)
    html = response.read()
    soup = bs(html,'html.parser')
    rawmeat = soup.findAll('a',attrs={'class':'next'})

###3.判断某url下一页是否存在

def CheckNum(url):
        response = urllib2.urlopen(url)
        html = response.read()
        soup = bs(html, 'html.parser')
        rawmeat = soup.findAll('a', attrs={'class': 'next'})
        if len(rawmeat) == 0:
            flag = 1
        else:
            flag = 0
        return flag

def GetName(url,filename):
    f = open(r'cookie.txt', 'r')
    tieba_cookie = {}
    for line in f.read().split(';'):
        name, value = line.strip().split('=', 1)
        tieba_cookie[name] = value

    tieba = requests.get(url=url, cookies=tieba_cookie).text
    soup = bs(tieba, 'lxml')
    raw_user = soup.find_all('span',attrs={'class':'name'})
    file = open(filename,'a+')
    for i in raw_user:
        file.write(i.a.text + '\r\n')
